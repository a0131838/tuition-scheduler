# 小赵批准执行队列 v1

用途：老板不在电脑旁时，由小赵把“已批准但待 Codex 执行”的事项写入固定队列；Codex 回到桌面后读取并执行。

## 固定队列文件

`ops/codex/approved-execution-queue.json`

## 小赵写队列规则

1. 只有老板明确说出以下口令之一，才允许写入队列：
   - `这条我批准，加入执行队列`
   - `记录到执行队列，待 Codex 执行`
2. 小赵只负责写入队列，不负责直接改代码或部署。
3. 同一事项若已在队列中且状态仍为 `pending`，优先更新备注，不重复新建。
4. 写入队列后，必须用老板可读格式回复：
   - 已加入执行队列
   - 队列编号
   - 事项摘要
   - 待 Codex 执行动作
5. 未经老板批准，不得把“建议”或“预览”直接升为执行队列事项。

## 队列条目最小字段

1. `queueId`
2. `createdAt`
3. `approvedBy`
4. `sourceChannel`
5. `summary`
6. `requestedAction`
7. `constraints`
8. `status`
9. `refs`

## 状态定义

1. `pending`：老板已批准，待 Codex 执行
2. `executing`：Codex 正在执行
3. `done`：Codex 已完成
4. `cancelled`：老板撤回或不再执行

## Telegram 推荐口令

1. `小赵，这条我批准，加入执行队列。`
2. `小赵，记录到执行队列，待 Codex 执行。`
3. `小赵，列出当前执行队列。`

## Codex 固定口令

1. `读取执行队列`
2. `执行第1条`
3. `执行所有已批准提案`

## 默认原则

1. 队列只承接“老板已批准”的事项。
2. 小赵写队列，Codex 执行，职责分离。
3. Codex 执行前仍需做影响评估；若会影响其他功能，先回报再执行。
