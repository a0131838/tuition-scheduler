# 小赵写操作分级开放清单 v1

更新日期：2026-03-05
目标：在不影响现有业务逻辑的前提下，分阶段放开小赵写操作。

## 1. 开放分级

### L0（已开放，低风险可逆）

1. `ticket.append_followup_note`
- 范围：仅追加工单备注
- 约束：`maxAffected <= 1`
- 风险：低

2. `package_txn.update_note_only`
- 范围：仅改流水备注
- 约束：不得改 `deltaMinutes/kind/createdAt`
- 风险：低

### L1（建议下一批开放）

1. `attendance.update_status`（单条）
- 场景：误点名修正
- 约束：单条、同日、可回滚
- 风险：中

2. `session.replace_teacher_single`（单条）
- 场景：临时代课
- 约束：不得改时间与班级
- 风险：中

### L2（需主管审批）

1. `partner_settlement.mark_exported`
- 约束：必须有导出前检查结果
- 风险：中高

2. `receipt_approval.update_status`
- 约束：必须保留审批链审计
- 风险：中高

### L3（暂不开放自动写）

1. 金额、分钟数直接改写
2. 跨月历史批量修正
3. 批量删除
4. 绕过业务规则的直接 SQL

## 2. 每一级共同门槛

1. 先 Preview 再 Apply。
2. Apply 必须确认短语匹配。
3. `affectedCount <= maxAffected`。
4. 必须有回滚入口。
5. 执行后必须 Post-check。

## 3. 验收标准（每个任务类型）

1. 连续 20 单预览正确率 100%。
2. 连续 20 单执行后核验通过率 100%。
3. 零越权字段修改。
4. 零未确认执行。
