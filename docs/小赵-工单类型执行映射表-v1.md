# 小赵工单类型执行映射表 v1

用途：规定每种工单类型，小赵该查什么、输出什么、能否自动预览、是否允许确认执行。

说明：
1. 这里只写最小执行剧本，不写长 SOP。
2. `可预览` 不等于 `可直接执行`。
3. 涉及财务金额、课包创建、结算状态的，一律先人工判断。

| 工单类型 | 目标 | 必查项 | 输出 | 可预览 | 可确认执行 |
| --- | --- | --- | --- | --- | --- |
| 改课程时间 | 给出现有课次可改方案 | 学生、老师、原课次、老师冲突、教室冲突、可约时间 | 可改时间方案、阻塞项、建议动作 | 是 | 否 |
| 改上课老师 | 给出可换老师方案 | 原课次、候选老师、老师冲突、学科匹配、地点 | 可替换老师、阻塞项、建议动作 | 是 | 否 |
| 临时取消&请假课程 | 预览取消影响 | 目标课次、取消对象、是否收费、是否点名 | 取消影响、收费提示、需确认项 | 是 | 否 |
| 补课加课 | 给出补/加课预览 | 课包是否足够、老师可用、教室/地点、时间 | 可排方案、阻塞项、建议动作 | 是 | 否 |
| 新学生购买课时包 | 整理待补信息 | 学生、来源、年级、课程、负责人 | 缺失信息、下一步人工动作 | 否 | 否 |
| 新排课 | 给出新排课方案 | 课包/评估课规则、老师可用、地点、时间 | 可排方案、阻塞项、建议动作 | 是 | 否 |
| 临时评估学生（没有买课程） | 给出评估课方案 | 学生、老师、地点、时间、评估课免扣规则 | 可排评估课方案、后续点名要求 | 是 | 否 |
| 评估学生（已买课程） | 给出评估课方案 | 学生、课包、老师、地点、时间 | 可排方案、是否免扣建议 | 是 | 否 |
| 学术问题 | 转为人工处理建议 | 学生、课程、老师、问题内容、证据 | 风险摘要、建议负责人、建议跟进动作 | 否 | 否 |
| 非学术问题 | 转为人工处理建议 | 学生、问题内容、来源、证据 | 风险摘要、建议负责人、建议跟进动作 | 否 | 否 |

## 当前建议首批开放

只开放到“可预览”：

1. 改课程时间
2. 补课加课
3. 新排课
4. 临时评估学生（没有买课程）
5. 评估学生（已买课程）

## 当前不建议开放自动执行

1. 新学生购买课时包
2. 学术问题
3. 非学术问题
4. 任何涉及金额、课包新建、结算状态的动作

## 默认原则

`先把工单处理成“可执行预览”，不要直接把工单处理成“已执行结果”。`
