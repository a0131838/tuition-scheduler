# Tuition Scheduler 主管版手册（风控 SOP）

更新日期：2026-03-05  
适用对象：主管、管理者、owner manager  
口径来源：`docs/员工使用手册-完整版.md`

## 1. 主管目标

1. 确保教学执行不断链（排课、点名、反馈）。
2. 确保财务数据可追溯（扣减、审批、结算、工资）。
3. 确保团队按统一流程执行。

## 2. 每日三段 SOP

### 2.1 开工检查（10-15 分钟）
1. `/admin/todos`：未点名、冲突、续费预警
2. `/admin/conflicts`：当日冲突优先清零
3. 指派责任人和完成时限

### 2.2 中午复查（5 分钟）
1. 复查点名与反馈完成率
2. 复查高风险学生（余额告急、排课密集）

### 2.3 收工复盘（10 分钟）
1. 看未闭环事项
2. 财务抽样核对（ledger/审批/报表）
3. 明日重点事项排程

## 3. 风险分级

### P0（立即处理）
- 大面积未点名
- 严重冲突未处理
- 财务数据明显不一致

### P1（当日闭环）
- 单点排课失败
- 票据审批卡住
- 工单超时未处理

### P2（本周优化）
- 流程重复手工操作过多
- 报表口径不统一
- 文档与功能不一致

## 4. 管理权限治理

- 页面：`/admin/manager`、`/admin/manager/users`
- 原则：最小权限、关键账号保护、所有高风险操作留痕。

## 5. OpenClaw 小赵监管规范

1. 所有数据变更必须先 Preview。
2. 主管确认后才能 Apply。
3. 批量/删除/财务变更必须有回滚方案。
4. 每次执行必须有“证据输出”。
