# Tuition Scheduler 主管版手册（风控SOP）

更新日期：2026-02-09
适用对象：运营主管、校区负责人、owner/manager

## 1. 目标
- 保证日常排课连续性
- 控制冲突、未点名、反馈超时、课包耗尽风险
- 保证员工按统一流程执行

## 2. 每日SOP（主管）

### 2.1 开工检查（10-15分钟）
1. 打开 `待办中心`（`/admin/todos`）
2. 核查四个指标：
- 今日未点名数
- 昨日遗留未点名数
- 冲突巡检问题数
- 续费预警人数
3. 若冲突巡检异常：点击 `立即复检`
4. 必要时执行 `自动修复`（老师冲突）

### 2.2 中午复查（5分钟）
- 检查今日课次未点名是否下降
- 检查老师端反馈是否有超时趋势

### 2.3 收工复盘（10分钟）
- 确认明日提醒（老师/学生）已处理
- 确认高风险学生已建立续费跟进任务
- 确认当天关键变更（换老师/取消）有记录

## 3. 风险分级与处置

### 3.1 P0（立即处理）
- 大面积课次冲突（老师/教室）
- 今日大量未点名
- 登录与核心业务不可用

处置：
1. 立刻暂停新增排课
2. 优先用 `冲突处理中心` 逐条落地修复
3. 必要时执行自动修复并复检
4. 通知负责人

### 3.2 P1（当日闭环）
- 单个老师连续反馈超时
- 明日提醒未确认
- 关键学生无有效课包但需排课

处置：
1. 指派专人处理
2. 2小时内回报处理结果
3. 当日收工前复核

### 3.3 P2（本周优化）
- 数据录入不规范（课程、级别、老师能力）
- 文案语言不统一
- 导出模板细节优化

## 4. 关键规则（管理视角）
- 报名前必须有有效课包。
- 同一学生同一课程不重复建立报名关系。
- 同一学生可同时持有班课包和一对一包。
- 课包同模式有效期不允许重叠。
- 可上课时间仅 `08:00-23:00`。

## 5. 管理者权限治理
- 页面：`/admin/manager/users`
- 要点：
- 普通管理者不能编辑 `zhao hongwei`
- manager-admin 受保护不可编辑
- owner 负责维护 manager 名单

执行建议：
1. 每周一次账号盘点（离职账号、弱口令、角色错误）
2. 新员工入职当天完成账号创建+最小权限分配
3. 重要账号密码重置后通知重新登录

## 6. 周度运营看板建议
- 本周总课次/完成课次
- 未点名率
- 冲突发生率（老师冲突、教室冲突）
- 反馈按时提交率
- 续费预警转化率

## 7. 月度审计清单
1. 随机抽查 20 个课次：点名与扣费一致性
2. 抽查 20 个学生：课包余额与排课记录一致性
3. 抽查导出文件：中文显示、字段完整性
4. 复核系统用户权限与账号状态

## 9. 新员工培训SOP（主管执行）
1. 账号与登录（5分钟）
2. 待办中心操作（15分钟）
3. 学生详情快速排课（20分钟）
4. 报名与课包规则（20分钟）
5. 冲突处理演练（20分钟）
6. 导出文件复核（10分钟）

考核标准：
- 能独立完成 1 次从报名到排课再到点名的完整流程
- 能识别并处理常见三类报错
- 能按SOP完成每日巡检
