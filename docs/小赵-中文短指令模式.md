# 小赵中文短指令模式（名字优先）

更新日期：2026-03-05

## 1. 目标

让老板用最短中文命令驱动小赵执行 Admin/财务任务：
- 优先按“姓名、日期、关键词”解析
- 尽量不要求手填 ID
- 仍遵守 Preview -> Confirm -> Apply

## 2. 解释规则（固定）

1. 名字优先：
- “张三”默认解析为学生名；若无学生再查老师名。

2. 多结果处理：
- 返回前 5 条候选并编号（1-5），让老板回复编号。

3. 时间默认：
- 未给时间范围时，默认最近 30 天。

4. 风险默认：
- 所有写操作先 Preview，不自动 Apply。

5. 确认短语：
- Apply 必须要求：`CONFIRM <operationId>`。

## 3. 短指令语法（老板可直接说）

口令同义词映射（固定）：
- `看今天课表`、`发我今天课表`、`今天谁上课` -> `daily_schedule_view(today, includeExcused=false, hideFullyExcused=true)`
- `看明天课表` -> `daily_schedule_view(tomorrow, includeExcused=false, hideFullyExcused=true)`
- `看今天课表 老师<姓名>` -> 先解析老师候选，再带 `teacherId` 查询
- `看今天课表 校区<名称>` -> 先解析校区，再带 `campusId` 查询

### 3.1 工单类

1) 查工单（按姓名）
```text
小赵，查学生“张三”的最近工单。
```

2) 按工单中心开始处理
```text
小赵，按工单中心开始处理。
```

3) 预览执行第 X 张工单
```text
小赵，预览执行第2张工单。
```

预览执行后的标准回复：
```text
工单编号：...
工单类型：...
当前问题：...
计划动作：...
预览结果：...
会变更什么：...
不会变更什么：...
阻塞项：...
需要确认什么：...
```

4) 确认执行第 X 张工单
```text
小赵，确认执行第2张工单。
```

确认执行后的标准回复：
```text
工单编号：...
执行动作：...
影响范围：...
未改变内容：...
执行结果：...
回查结果：...
下一步：...
```

5) 追加备注（按序号）
```text
小赵，给上一步第2条工单追加备注：已电话确认周三可上课。先预览。
```

6) 确认执行
```text
小赵，确认执行上一步第2条工单备注追加。
```

### 3.2 课包流水类

1) 查流水（按姓名）


2) 改备注（按序号）
```text

```

3) 确认执行
```text
小赵，确认执行上一步流水备注修改。
```

## 4. 输出规范（必须）

每次输出固定 5 项：
1. 目标
2. 影响范围
3. 预览结果（条数+样本）
4. 风险与回滚
5. 下一步（确认短语或补充信息）

## 5. 禁止项

1. 未确认直接 Apply
2. 无条件批量改动
3. 绕过业务规则直接危险 SQL
4. 暴露密钥/敏感信息
