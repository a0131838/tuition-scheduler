# Tuition Scheduler 员工使用手册（标准版）

更新日期：2026-03-05  
适用对象：前台、教务、学管、财务、主管、老师  
口径声明：本手册与《员工使用手册-完整版》一致；如有冲突，以完整版为准。

主文档：`docs/员工使用手册-完整版.md`

---

## 1. 每日先后顺序（所有岗位共识）

1. 先看 `/admin/todos`
2. 再清 `/admin/conflicts`
3. 再做 `/admin/schedule` 与 `/admin/schedule/new` 排课改课
4. 最后处理报表、导出、票据与结算

---

## 2. 岗位流程

### 2.1 前台顾问
1. `/admin/students` 建档
2. 检查课包是否有效（无包先交学管）
3. `/admin/booking-links` 发预约链接
4. 跟进审批落课结果

### 2.2 教务排课
1. `/admin/todos` 清未点名
2. `/admin/conflicts` 清冲突
3. `/admin/schedule` 调整课次
4. `/admin/classes/[id]/sessions` / `/admin/students/[id]` 做班课与1v1排课

### 2.3 学管
1. `/admin/packages` 管理课包
2. `/admin/packages/[id]/ledger` 对账
3. `/admin/students/[id]` 复核剩余与未来课次

### 2.4 财务
1. `/admin/receipts-approvals` 审批票据
2. `/admin/packages/[id]/billing` 管票据关联
3. `/admin/reports/partner-settlement` 与 `/billing` 做结算
4. `/admin/reports/teacher-payroll` 复核工资

### 2.5 老师
1. `/teacher` 看今日/明日课
2. `/teacher/availability` 维护可上课时间
3. `/teacher/sessions/[id]` 点名与反馈
4. `/teacher/student-feedbacks` 补反馈

---

## 3. 核心规则（必须遵守）

1. 无有效课包，不允许排课/报名。
2. 排课必须满足：老师可用 + 老师无冲突 + 教室无冲突。
3. 冲突必须在 `/admin/conflicts` 闭环。
4. 点名与课包扣减必须一致可追溯。
5. 财务导出前必须确认审批链完整。

---

## 4. 常见异常

1. `No active package for this course`：先建/续包。
2. `No eligible teachers`：查授课能力、可用时间、冲突。
3. `Outside availability`：调整可用时间或改时段。
4. `Teacher/Room conflict`：去冲突中心处理。
5. 老师端无课次：检查老师账号绑定。

---

## 5. OpenClaw 小赵代操作要求

1. 数据修改统一两阶段：`Preview -> 你确认 -> Apply`。
2. 每次必须汇报：影响条数、变更前后、未变更项、验证证据。
3. 高风险操作（删除/批量财务/批量排课）先给回滚方案。
4. 一律优先走业务 API 与现有流程，不做破坏性绕库操作。
