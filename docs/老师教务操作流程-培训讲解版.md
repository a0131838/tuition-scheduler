# 老师 + 教务 操作流程（培训讲解版）

更新日期：2026-02-14  
培训时长：30分钟  
对象：新入职教务、班主任、老师

## 1. 培训目标（2分钟）
- 明确教务和老师在系统内的边界与交接点。
- 能独立完成“排课-上课-点名-反馈-复核”闭环。
- 知道高频报错的处理路径。

## 2. 系统入口与角色（3分钟）
- 教务：`/admin`（登录：`/admin/login`）
- 老师：`/teacher`
- 首启管理员：`/admin/setup`

讲解重点：
- 教务负责“排课与风险控制”。
- 老师负责“执行与反馈闭环”。

## 3. 教务核心流程演示（10分钟）

### 3.1 开工检查（2分钟）
演示页面：`/admin/todos`
- 清未点名
- 看冲突
- 看明日提醒
- 看续费预警

口令：先风险后排课。

### 3.2 排课调课（4分钟）
演示页面：`/admin/schedule`、`/admin/schedule/new`
- 在周课表换老师/删课
- 新建单次课与预约
- 失败时回冲突中心处理

### 3.3 冲突闭环（2分钟）
演示页面：`/admin/conflicts`
- 先处理 appointment
- 再处理 session
- 处理完回周课表复核

### 3.4 学员与课包联动（2分钟）
演示页面：`/admin/students`、`/admin/packages`
- 排课前必须先确认课包有效
- 扣费争议按“点名页 → ledger”排查

## 4. 老师核心流程演示（8分钟）

### 4.1 课前（3分钟）
演示页面：`/teacher`、`/teacher/availability`
- 确认今日/明日课程
- 维护未来2-4周可上课时间

### 4.2 课后（3分钟）
演示页面：`/teacher/sessions`、`/teacher/sessions/[id]`
- 点名
- 反馈

### 4.3 老师资料（2分钟）
演示页面：`/teacher/card`
- 更新介绍
- 导出名片

## 5. 异常处理演练（5分钟）

### 5.1 报错1：无有效课包
- 现象：`Student has no active package for this course`
- 演练：先建/续课包，再排课

### 5.2 报错2：无可用老师
- 现象：`No eligible teachers`
- 演练：查授课能力 + availability + 冲突

### 5.3 报错3：超出可用时间
- 现象：`Outside availability`
- 演练：调课时间或先维护老师可用时段

## 6. 培训后考核（2分钟）
- 教务实操：
  1. 完成1次调课
  2. 完成1次冲突处理
  3. 完成1次新建单次课
- 老师实操：
  1. 确认今日课程
  2. 完成1条点名
  3. 完成1条反馈

通过标准：
- 独立完成且无关键错误（错老师、错教室、漏点名）。

## 7. 培训讲师话术建议
- “所有冲突都进冲突中心，不在别处硬改。”
- “未点名是最高优先级，先清再排课。”
- “课包有效性是排课前置条件。”
