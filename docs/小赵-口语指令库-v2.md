# 小赵口语指令库 v2（老板专用）

更新日期：2026-03-05

## 1. 一句话查数据

1. `小赵，查学生“张三”的最近工单。`
2. `小赵，查学生“张三”最近30天课包流水。`
3. `小赵，查老师“李老师”明日冲突。`
4. `小赵，给我今天财务风险前三项。`
5. `小赵，看今天课表。`
6. `小赵，看今天课表（老师：李老师）。`
7. `小赵，看今天课表（校区：Bukit Timah）。`

## 2. 一句话做预览

1. `小赵，把“张三”最近工单里最新一条追加备注：已联系家长确认周三。先预览。`
2. `小赵，把“张三”最近流水中第1条备注改为：财务复核通过。先预览。`
3. `小赵，预览执行第2张工单。`

说明：
1. 工单预览必须用业务语言回复。
2. 不要给老板看 `validate-op-request`、JSON 文件名、脚本路径。
3. 预览重点是：会改什么、不改什么、阻塞项、下一步确认点。

## 3. 一句话确认执行

1. `小赵，确认执行：CONFIRM <operationId>`
2. `小赵，确认执行第2张工单`

说明：`confirmationText` 必须与 `confirmPhrase` 完全一致。

## 4. 每日节奏命令

1. 开工：`小赵，开工巡检。`
2. 中午：`小赵，中午复查。`
3. 收工：`小赵，收工复盘。`
4. 日报：`小赵，发今天的系统风控日报。`
5. 员工遗漏：`小赵，发今天员工遗漏追踪清单。`

## 4.1 问题与审计命令

1. `小赵，记录这个问题。`
2. `小赵，做本周系统审计。`
3. `小赵，给我系统优化提案。`
4. `小赵，更新这个规则到主文档。`

## 4.1A 批准执行队列命令

1. `小赵，这条我批准，加入执行队列。`
2. `小赵，记录到执行队列，待 Codex 执行。`
3. `小赵，列出当前执行队列。`
4. `小赵，读执行队列。`

说明：
1. 这三条只负责入队和查看，不直接执行代码或部署。
2. 小赵入队后，Codex 回到桌面再读取并执行。
3. 队列文件固定为 `ops/codex/approved-execution-queue.json`。

## 4.1B 老板汇总命令

1. `小赵，今天工单最急的3张。`
2. `小赵，今天谁还没处理工单。`
3. `小赵，今天排课风险汇总。`
4. `小赵，今天财务异常汇总。`

说明：
1. 这四条都是短汇总，不展开技术过程。
2. 默认只报今天最重要的结论、前三项风险、负责人和下一步。
3. `给我系统优化提案` 必须按审批格式输出：事项、问题、影响、建议方案、风险等级、是否建议改代码、优先级、是否可直接批准。

## 4.2 工单驱动命令

1. `小赵，按工单中心开始处理。`
2. `小赵，预览执行第2张工单。`
3. `小赵，确认执行第2张工单。`

## 5. 小赵标准回报模板

1. 任务目标
2. 影响范围
3. 执行结果
4. 风险与回滚
5. 下一步建议

补充口径：
1. 老板默认看的是业务结论，不是技术过程。
2. 查询类先回答“查到了什么”。
3. 日报/巡检类先回答“今天最重要的是什么”。
4. 预览类先回答“会改什么、不改什么、还缺什么确认”。
5. 执行类先回答“已经改了什么、结果是否成功”。
6. 除非老板点名要看技术细节，否则不要输出内部文件名、脚本名、JSON 名、validate 结果。

## 6. 失败兜底回复规范

1. 多个同名：返回候选编号让老板选 1 个
2. 无数据：只回 `无匹配学生` 或 `无匹配工单`
3. 无通道：提示可执行命令，不让老板手写 SQL

## 7. “看今天课表”固定执行规则（新增）

1. 默认解释：
- `看今天课表` = 查询今日（本地时区）可见课表，按老师端可见口径过滤请假。

2. 默认参数：
- `date=today`
- `includeExcused=false`
- `hideFullyExcused=true`

3. 调用优先级：
- 首选 API：`GET /api/admin/ops/daily-schedule-view`
- 兜底脚本：`node --env-file=.env ops/codex/query-daily-schedule-view.mjs "YYYY-MM-DD" "<teacherId可空>" "<campusId可空>"`

4. 固定返回字段：
- 课次时间（startAt-endAt）
- 老师
- 校区/教室
- 可见学生名单
- 汇总：`visibleSessions`、`visibleStudentCount`

## 8. 频率约束（新增）

1. `记录这个问题`：实时触发，但先过问题入库规则。
2. `系统风控日报`：每天 1 次，不重复输出历史内容。
3. `员工遗漏追踪清单`：每天 1 次，按当天待闭环项输出。
4. `本周系统审计`：建议每周 2 次（周三中午、周日晚）。
5. `给我系统优化提案`：按需触发，按审批格式输出：事项、问题、影响、建议方案、风险等级、是否建议改代码、优先级、是否可直接批准。
6. `更新这个规则到主文档`：只在老板明确批准后执行。
7. `按工单中心开始处理`：按需触发，不主动干扰其他既有指令。
8. `加入执行队列`：按需触发，只在老板明确批准后写入。
9. `老板汇总命令`：按需触发，只报当天最重要信息，不做长文展开。
