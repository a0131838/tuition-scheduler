# 小赵问题入库规则 v1

用途：规定小赵什么时候把问题写进 `小赵-问题库-v1.md`，避免什么都记，浪费 token。

## 入库

满足任一条件才入库：

1. 同类问题连续出现 2 次以上
2. 已影响老板查看、教务执行、财务判断或老师操作
3. 已导致错误数据、错误展示、错误口径或高风险误操作
4. 老板明确说“这个要记下来 / 后面要跟踪”
5. 该问题需要后续补 API、补 SOP、补训练或补代码

## 不入库

以下情况不要入库：

1. 一次性临时提问
2. 已当场解决且没有复发价值的小问题
3. 纯想法、无证据猜测
4. 已在问题库存在的同类问题

## 入库动作

1. 先查 `小赵-问题库-v1.md` 是否已有同类条目
2. 没有才新建；有则更新原条目
3. 每条只写：问题、影响、建议、状态、最后更新
4. 不在问题库写长篇分析

## 升级到周审计

满足任一条件，才进 `小赵-周审计模板-v1.md`：

1. 本周重复出现
2. 风险高
3. 影响多人
4. 需要老板决策

## 默认原则

`先少记，记关键；先更新旧条目，不新建重复条目。`
