# 岗位SOP：教务排课

适用页面：`/admin/todos`、`/admin/schedule`、`/admin/schedule/new`、`/admin/conflicts`、`/admin/sessions/[id]/attendance`

## 每日开工顺序
1. `待办中心`：清昨日/今日未点名。
2. `冲突巡检`：有冲突先处理再新增排课。
3. `周课表`：处理改课、换老师、换教室。
4. `新建(单次)`：补加课次/预约。

## 排课规则
- 课次时间必须在老师可上课时间内（08:00-23:00）。
- 同时校验老师冲突、教室冲突、预约冲突。
- 冲突统一在 `冲突处理中心` 闭环。

## 点名规则
- 点名保存会自动扣减课包并记录流水。
- `EXCUSED` 是否扣费要按实际业务选择。
- 发现扣费异常先查课次点名页和课包流水。

## 异常处理
- `Outside availability`：去老师可用时间页维护。
- `No eligible teachers`：检查老师授课能力和时间。
- 课包不足：联系学管先处理续费再排课。

## 每日收工检查
- 未点名是否清零
- 冲突是否清零
- 明日课程是否可执行
