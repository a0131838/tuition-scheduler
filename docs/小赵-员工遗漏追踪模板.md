# 小赵员工遗漏追踪模板

更新日期：2026-03-05
目标：每天自动识别“谁可能忘了做什么”，用于主管追办。

## 1. 遗漏判定规则（v1）

1. 工单遗漏
- 条件：`nextActionDue < 当前时间` 且工单未关闭
- 首选数据源：`/api/admin/ops/ticket-followups-overdue`
- 归属：`owner`

2. 教务遗漏
- 条件：课程结束超过 `3小时` 后仍存在 `UNMARKED`
- 首选数据源：`/api/admin/ops/unmarked-followups-overdue`
- 归属：老师姓名（session.teacher 或 class.teacher）

3. 财务复核遗漏
- 条件：当日出现 `PackageTxn.kind in (ADJUST, ROLLBACK)` 且无后续复核记录
- 归属：财务值班负责人（由老板或主管指定）

## 2. 输出模板

```text
【员工遗漏追踪】<日期>

A. 工单遗漏（按人）
- <ownerA>: <N> 项（最早超时: <时间>）
- <ownerB>: <N> 项

B. 教务遗漏（按老师）
- <teacherA>: <N> 节未点名
- <teacherB>: <N> 节未点名

C. 财务复核遗漏
- ADJUST: <N> 条
- ROLLBACK: <N> 条

D. 追办建议
- 今天 <HH:mm> 前：<ownerA> 关闭前 <K> 项超时工单
- 今天 <HH:mm> 前：<teacherA> 补点名 <N> 节
- 今天 <HH:mm> 前：财务复核 ADJUST/ROLLBACK 前 <K> 条
```

## 3. 主管使用口令

```text
小赵，生成“员工遗漏追踪清单”，按严重度排序，并给每人今天截止时间建议。
```
