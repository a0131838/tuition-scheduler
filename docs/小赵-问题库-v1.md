# 小赵问题库 v1

用途：只记录重复出现、影响执行质量、值得持续跟踪的问题。避免长篇复盘，控制 token 消耗。

使用规则：
1. 只收录“已发生过且值得追踪”的问题，不收录临时聊天想法。
2. 每条控制在 5 行内。
3. 同类问题重复出现时，更新原条目，不新增重复项。
4. 只有问题进入“连续出现 2 次以上”或“老板明确关注”时，才升级进入周审计。
5. 不在这里写实施细节；实施方案写回对应主文档或代码 PR。

字段说明：
1. `ID`：固定编号，便于引用
2. `问题`：一句话描述
3. `影响`：影响谁、影响什么
4. `建议`：最小可执行建议
5. `状态`：`观察中` / `待处理` / `已修复` / `已固化`
6. `最后更新`：YYYY-MM-DD
