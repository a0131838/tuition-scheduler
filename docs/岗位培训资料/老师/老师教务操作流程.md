# 老师 + 教务 操作流程（详细版）

更新日期：2026-03-05  
口径：与 `docs/员工使用手册-完整版.md` 一致。

## 1. 教务日常 SOP（按时间）

### 1.1 开工 10 分钟
1. `/admin/todos` 清昨日/今日未点名
2. `/admin/conflicts` 先清冲突
3. 检查明日提醒与续费预警

### 1.2 排课与调课
1. `/admin/schedule` 查看周课表
2. `/admin/schedule/new` 建单次课/预约
3. `/admin/classes/[id]/sessions` 维护班级课次
4. `/admin/students/[id]` 快速排1v1

### 1.3 收工 10 分钟
1. 再看 `/admin/todos`
2. 处理遗漏点名/反馈
3. 留痕异常与次日待办

## 2. 老师日常 SOP

### 2.1 课前
1. `/teacher` 确认今日/明日课
2. `/teacher/availability` 维护未来 2-4 周可上课时间
3. `/teacher/alerts` 看提醒

### 2.2 课后
1. `/teacher/sessions/[id]` 完成点名
2. 提交反馈（12 小时内）
3. `/teacher/student-feedbacks` 复核未完成项

## 3. 协作闭环

1. 教务排课前先看冲突中心。
2. 老师异常（请假/缺课）即时反馈教务。
3. 教务处理后在周课表复核结果。
4. 财务核对点名与扣减一致性。

## 4. 常见异常

1. `No active package`：转学管建/续包。
2. `Outside availability`：补可用时间或改排课。
3. `Teacher/Room conflict`：去冲突中心闭环。
4. 老师端无课：检查账号绑定与课次老师字段。
