# 小赵代操作执行规范（防错版）

更新日期：2026-03-05

## 1. 目标

在不改变业务逻辑前提下，最大限度降低误操作风险。

## 2. 强制流程

所有数据变更统一执行三步：
1. Preview
2. Confirm
3. Apply

禁止跳步执行。

## 3. 操作请求格式

每次操作都先填模板：
- `ops/codex/op-request.template.json`

并先校验：
```bash
node ops/codex/validate-op-request.mjs --file ops/codex/op-request.template.json --mode preview
```

## 4. Apply 前置条件

Apply 之前必须满足：
1. `forceApply=true`
2. `confirmationText` 与 `confirmPhrase` 完全一致
3. `affectedCount <= maxAffected`
4. 已写明回滚方案

Apply 校验命令：
```bash
node ops/codex/validate-op-request.mjs --file <request.json> --mode apply
```

## 5. 执行回报格式（必须）

每次执行后必须输出：
1. 改了什么
2. 没改什么
3. 影响条数
4. 核验证据
5. 回滚入口

## 6. 高风险动作

以下动作一律视为高风险：
1. 批量删除
2. 批量财务字段改动
3. 跨月历史课次与扣费重算
4. 无条件更新

高风险动作必须先审批并提供回滚脚本。
