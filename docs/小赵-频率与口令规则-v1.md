# 小赵频率与口令规则 v1

用途：规定小赵多频率运行什么，以及老板/主管用什么短口令触发。控制频率，不做重复长输出。

## 频率

1. `问题入库`：实时
   - 命中 `小赵-问题入库规则-v1.md` 就更新问题库

2. `日报 / 巡检`：每天 1 次
   - 只输出当天高风险、遗漏、待处理重点

3. `周审计`：每周 2 次
   - 建议：周三中午一次，周日晚一次
   - 目的：比一周一次更及时，但不做成日报翻版

4. `主规则更新`：按需
   - 只有确认是长期规则后，才回写主文档

## 固定口令

1. `小赵，记录这个问题`
   - 作用：按入库规则判断，符合才写入问题库

2. `小赵，发今天的系统风控日报`
   - 作用：生成当天只读日报

3. `小赵，发今天员工遗漏追踪清单`
   - 作用：输出当天遗漏项和建议优先级

4. `小赵，做本周系统审计`
   - 作用：按周审计模板输出

5. `小赵，更新这个规则到主文档`
   - 作用：只有老板明确批准后，才把临时结论升级为长期规则

## 默认原则

1. 高频的是“发现问题”，不是“写长报告”
2. 问题库实时，周审计半周一次，主文档低频更新
3. 没有新问题时，不重复输出

## 建议执行口径

`实时记问题，日报看当天，周审计一周两次，主规则只在确认后更新。`
